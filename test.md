# 线扫相机算法工程师入职准备与学习清单

> 背景：原机器视觉从业者，熟悉 Halcon + C++ 算法封装；新公司做仿 LMI（Gocator / GoPxL）的线扫相机，产品形态以**网页端配置与可视化**为主。

---

## 一、先建立产品心智模型（对照 LMI）

LMI 这类产品的核心不是“拍一张图再离线处理”，而是：

**传感器/控制器 + 边缘算法 + Web UI + SDK/API**，一套在线测量闭环。

建议尽快搞清你们产品对应 LMI 的哪些能力：

| LMI 概念 | 你需要理解的含义 | 对你岗位的意义 |
|---------|------------------|---------------|
| 激光线轮廓传感（Line Profiler） | 激光三角测量 → 单帧 Profile（X/Z） | 算法输入形态从 2D 图像变为轮廓/点云 |
| Encoder / Trigger | 编码器同步、外触发、线速度与分辨率 | 决定 Y 方向采样与标定 |
| Profile → Surface | 多条轮廓拼接成表面（3D） | 测量工具的数据基础 |
| GoPxL Web UI | 浏览器配置曝光、ROI、工具、输出 | 算法参数如何被前端暴露 |
| REST API（配置） | JSON 读写参数 | 你封装的能力如何被调用 |
| GDP / 数据流协议 | 实时推送 Profile/Surface/测量结果 | 实时性、带宽、序列化 |
| Measurement Tools | Gap、Height、Section、平面拟合等 | 你最核心的算法交付物 |
| PLC / I/O / 工业协议 | OK/NG、数字量、EtherNet/IP 等 | 产线落地边界 |

**入职第一周建议动作：**

1. 用浏览器完整走一遍竞品或自家 Demo：连接 → 调光 → 标定 → 跑工具 → 看结果输出。
2. 画一张你们系统的数据流图：相机 → 采集 → 算法 → WebSocket/REST → 前端 / PLC。
3. 问清楚：算法跑在设备端、工控机，还是两者都有？延迟与帧率指标是多少？

## 二、你该准备的事情（入职前后）

### 1. 知识与资料准备

- [ ] 通读 LMI Gocator / GoPxL 用户手册中的：触发、编码器、曝光、标定、工具、输出
- [ ] 了解激光三角测量原理、景深、反射材质对轮廓质量的影响
- [ ] 复习点云/轮廓基础：滤波、分割、拟合、截面测量、坐标系变换
- [ ] 准备英文技术阅读习惯（手册、协议、Issue 讨论）
- [ ] 整理自己过去 Halcon 项目：曝光控制、标定、测量工具、封装接口设计经验（面试/入职讲解用）

### 2. 环境与工具准备

- [ ] 熟悉 Git（分支、PR、Code Review）——比 Halcon 脚本协作更常见
- [ ] 装好常用 IDE：VS / CLion（C++）、VS Code（前端/脚本）
- [ ] 了解 Docker 基础（很多 Web/后端服务会容器化）
- [ ] 准备抓包与调试：Wireshark、浏览器 DevTools、Postman/Insomnia
- [ ] 若有设备：学会用 IP 访问 Web UI、改网段、看日志

### 3. 入职后尽快对齐的问题（建议直接问同事）

- [ ] 产品对标 LMI 的哪一代能力？已实现 / 在研 / 不做分别是什么？
- [ ] 算法模块边界：谁负责采集驱动、谁负责测量工具、谁负责 Web 接口？
- [ ] 数据格式：Profile/Surface 内存布局、单位（mm）、坐标系约定
- [ ] 实时指标：最大线速率、分辨率、端到端延迟、丢帧策略
- [ ] 你们用的技术栈：C++ / C# / Python？前端 Vue / React？通信 REST + WS？
- [ ] 标定流程：工厂标定 vs 现场标定，谁写工具链？
- [ ] 交付形式：动态库、进程服务、还是脚本插件？

## 三、需要学习的内容（按优先级）

### P0：立刻要会（1–2 周建立可用能力）

#### A. 线扫 / 3D 轮廓传感本身

- [ ] 激光三角测量：基线、工作距、FOV、Z 分辨率与噪声来源
- [ ] Profile 数据：有效点、无效点、多峰、曝光不足/过曝
- [ ] 编码器模式 vs 时间触发：Y 分辨率如何算
- [ ] 从 Profile 拼 Surface：运动方向、抖动、拼接误差
- [ ] 常见缺陷：镜面反光、黑色吸光、台阶遮挡、多重反射

> 你的 Halcon 经验可迁移：滤波、拟合、阈值、几何测量；但要改成**以轮廓/点云为主**，而不是面阵图为主。

#### B. 测量算法（你的主责可能性最高）

- [ ] 轮廓预处理：去噪、缺口填补、ROI、坐标系对齐
- [ ] 几何工具：高度、宽度、间隙、角度、圆心、平面/直线拟合
- [ ] 截面与区域工具：指定 X 范围统计、模板比对、形变补偿
- [ ] 结果输出：数值 + OK/NG + 可视化叠加（前端要画的东西）
- [ ] 性能：单核/多核、SIMD、流水线，保证在线帧率

**Halcon → 自研的迁移建议：**

| 以前（Halcon） | 现在更常见 |
|----------------|------------|
| `measure_*` / `fit_*` | 自研 C++ 几何库或 Open3D / PCL / Eigen |
| HDevelop 调试 | 单元测试 + 离线回放数据集 |
| 算子封装给软件同事 | REST/SDK 接口 + 参数 JSON Schema |
| 图像窗口显示 | Web Canvas / WebGL 渲染轮廓与测量结果 |

#### C. 与 Web 产品协作的“接口思维”

- [ ] REST：用 JSON 读写曝光、触发、工具参数、Job/配方
- [ ] WebSocket 或二进制流：实时 Profile/Surface/结果
- [ ] 参数模型：哪些是设备级、扫描级、工具级（LMI 也是分层资源）
- [ ] 版本与配方：Job 保存/加载、参数校验、向前兼容

你不一定要成为前端专家，但必须能说清：

**“我的算法输入是什么、输出是什么、参数如何暴露、失败如何报错。”**

### P1：一个月内补齐（让你能独立扛模块）

#### D. 嵌入式 / 边缘软件基础（若算法在设备或工控机）

- [ ] Linux 基础：进程、线程、共享内存、网络、日志
- [ ] C++17/20 工程化：CMake、单元测试、静态检查
- [ ] 并发与实时：无锁队列、背压、丢帧策略
- [ ] 序列化：Protobuf / FlatBuffers / 自定义二进制（大数据量别只用 JSON）

#### E. 网页端相关（够用来联调即可）

- [ ] HTTP / WebSocket 原理
- [ ] 浏览器如何画实时曲线与伪彩高度图（Canvas/WebGL 概念）
- [ ] 前后端联调：CORS、鉴权、断线重连、背压
- [ ] （可选）Vue 或 React 基础，能读懂前端如何调你的 API

#### F. 工业现场与系统集成

- [ ] 编码器、光电、PLC 握手时序
- [ ] 数字 I/O、OK/NG、报警
- [ ] 常见工业协议概念：Modbus / EtherNet/IP / Profinet（先懂场景）
- [ ] 产线节拍、误检漏检、温漂与长期稳定性

### P2：进阶加分（3 个月成为骨干）

- [ ] 标定与精度：畸变、激光平面、手眼/世界坐标、重复精度评估
- [ ] 多传感器拼接 / 双目或双激光方案
- [ ] GPU / 加速（若点云很大）：CUDA 或专用 NPU
- [ ] 工具链产品化：工具市场、脚本扩展、自定义工具插件
- [ ] 对标竞品：Keyence、SICK、Cognex、SSZN 等线扫/轮廓方案差异
- [ ] 质量体系：算法验收标准、黄金样本集、回归测试平台

## 四、建议的 30 天学习路线

### 第 1 周：把产品跑通

- 完整使用 Web UI（或 LMI Demo）跑通一套测量
- 画清系统架构与模块边界
- 用 Postman 调通 2–3 个配置 API，抓一条数据流看格式

### 第 2 周：把数据吃透

- 离线保存 Profile/Surface 样本
- 用 C++ 写最小 Pipeline：读数据 → 滤波 → 拟合高度 → 输出 JSON
- 和前端约定一种可视化叠加协议（点、线、框、文本）

### 第 3 周：做一个可上线的小工具

- 例如：台阶高度 / 间隙 / 平面度（选一个你们最常用的）
- 补齐参数校验、异常码、单元测试、性能压测
- 接入配方保存与加载

### 第 4 周：现场化与稳定性

- 不同材质/速度下的鲁棒性
- 丢帧、断连、重连、日志
- 写清工具文档：原理、参数、适用场景、失败案例

## 五、你的优势与需要补的短板

### 优势（直接用）

- Halcon 测量与几何直觉强，转轮廓工具很快
- C++ 封装经验正好对应 SDK / 算法服务化
- 懂“给软件同事调用”——现在升级为“给 Web / PLC / 客户二次开发调用”

### 需要补的短板

- 从**面阵 2D + Halcon** → **线扫 Profile/Surface + 自研或轻量库**
- 从**本地库调用** → **REST + 实时数据流 + 网页可视化**
- 从**单机算法** → **在线系统：触发、编码器、节拍、稳定性**
- 从**算子调试** → **产品化：配方、版本、回归测试、文档**

## 六、推荐学习资源（精选）

### 产品与原理

- LMI Gocator / GoPxL 用户手册（触发、编码器、工具、SDK/REST 章节）
- 激光三角测量与线激光轮廓仪原理文章/白皮书
- 点云基础：法向、平面拟合、ICP（先概念后实现）

### 工程与算法库

- Eigen（线性代数 / 拟合）
- Open3D 或 PCL（点云处理，按项目选型）
- C++ 并发与性能（手写环形缓冲、Benchmark）

### Web 与接口

- REST API 设计基础 + JSON Schema
- WebSocket 实时推送
- （选学）Vue/React 官方入门，能读前端代码即可

### Cursor 使用（你刚接触）

- [ ] 用 Cursor 读公司代码：先问“这个模块的数据流是什么”
- [ ] 让 AI 帮忙写单元测试、解释陌生协议、生成接口文档草稿
- [ ] 规则：算法核心与数值正确性必须自己验证；AI 适合加速样板代码与排查
